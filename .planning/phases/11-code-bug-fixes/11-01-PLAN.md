## Phase 11: Code Bug Fixes

### Objectives
1. Fix HTTPException usage in backend services
2. Fix hardcoded path in frontend
3. Clean up duplicate code

### Context
Code review found several bugs that need fixing:
- HTTPException using positional args instead of keyword args (inconsistent with FastAPI best practices)
- Frontend uses Windows-style backslash in path which breaks on Linux
- Duplicate data fetching code between data_fetcher.py and fetch_data_service.py

### Requirements
- BUG-01: Fix HTTPException usage in prediction_service.py (positional → keyword args)
- BUG-02: Fix HTTPException usage in data_fetcher.py (positional → keyword args)
- BUG-03: Fix hardcoded backslash path in frontend app.py
- BUG-04: Remove duplicate/redundant code

### Tasks

#### Task 1: Fix HTTPException in prediction_service.py
**File:** `MarketSense-backend/app/services/prediction_service.py`

Change:
```python
# From (incorrect - positional):
raise HTTPException(400, "n_days must be positive")
raise HTTPException(400, f"Prediction not supported for {model.framework}")

# To (correct - keyword args):
raise HTTPException(status_code=400, detail="n_days must be positive")
raise HTTPException(status_code=400, detail=f"Prediction not supported for {model.framework}")
```
