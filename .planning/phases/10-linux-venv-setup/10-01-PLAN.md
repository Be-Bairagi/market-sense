## Phase 10: Linux venv Setup

### Objectives
1. Create venv_linux for MarketSense-backend
2. Create venv_linux for Marketsense-frontend

### Context
The project needs Linux-compatible virtual environments for both frontend and backend to run on Linux systems. The backend already has a venv_linux directory but may need to be recreated/reinstalled. The frontend needs a new venv_linux created.

### Requirements
- VENV-01: Create venv_linux for MarketSense-backend
- VENV-02: Create venv_linux for Marketsense-frontend

### Tasks

#### Task 1: Create venv_linux for MarketSense-backend
```bash
# Navigate to backend directory
cd MarketSense-backend

# Check if venv_linux exists
ls -la venv_linux

# If exists, remove and recreate
rm -rf venv_linux

# Create new virtual environment using Python 3.11
python3.11 -m venv venv_linux

# Activate and install dependencies
source venv_linux/bin/activate
pip install --upgrade pip
pip install -r requirements.txt

# Verify installation
python -c "import fastapi, yfinance, prophet; print('All packages installed successfully')"

# Deactivate
deactivate
```

#### Task 2: Create venv_linux for Marketsense-frontend
```bash
# Navigate to frontend directory
cd Marketsense-frontend

# Check existing venv
ls -la venv_311

# Create new venv_linux
python3.11 -m venv venv_linux

# Activate and install dependencies
source venv_linux/bin/activate
pip install --upgrade pip
pip install -r requirements.txt

# Verify installation
python -c "import streamlit, plotly, pandas; print('All packages installed successfully')"

# Deactivate
deactivate
```

### Verification
- [ ] venv_linux exists in MarketSense-backend/
- [ ] venv_linux exists in Marketsense-frontend/
- [ ] Backend packages (fastapi, yfinance, prophet) importable
- [ ] Frontend packages (streamlit, plotly, pandas) importable
- [ ] No installation errors

### Notes
- Use Python 3.11 (not 3.14) due to binary wheel availability
- Frontend already has venv_311 as reference
